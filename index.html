This page demonstrates how to structure a basic web page using HTML5 semantic tags. It is designed to be both accessible to screen readers and optimized for search engines.

HTML5 Semantic Tutorial

Why Use Semantic HTML?

Semantic HTML improves the meaning and structure of your content. Screen readers can easily interpret it, and search engines can better understand your page.

Key Semantic Elements

  • <header> – Represents the introductory content or navigation.
  • <nav> – Defines a section for navigation links.
  • <main> – Holds the main content unique to the page.
  • <section> – Groups related content.
  • <article> – Represents independent, self-contained content.
  • <footer> – Contains footer information like copyright.
